# bake

**bake** is a small asset-baking tool for C projects.

It reads a simple **INI description file** and generates a **C header** containing:

- interned string constants
- embedded file data
- stable numeric IDs
- optional enums and string arrays

The generated header can then be included directly in a program.

The tool was written as a **demo application for the STDX library**, exercising several modules such as the INI parser, filesystem utilities, IO helpers, logging, and string handling. The goal was to build a realistic program that validates the usability of the library while producing something genuinely useful. :contentReference[oaicite:0]{index=0}

---

# Command Line Usage

```
usage:
  bake.exe <input.ini> [-o out.h]

The input INI file describes strings and files to bake into a C header.

It accepts the following Top-level options (no section):
  guard          = DATA_H            # Include guard macro name (required)
  output         = out_file_name.h   # Optional; default is <ini filename>.h
  no_enum        = 0|1               # If 0, do NOT emit enums for each [section].
  no_strings     = 0|1               # If 0, do NOT emit const char* array for each [section].
  no_crc         = 0|1               # If 0, do NOT compute CRC32 for embedded file contents.
  bytes_per_line = N                 # Bytes per line for byte arrays (default: 8)
  comment        = "text..."          # Comment emitted at the top of the generated header
                                     # Supports \n and \t escapes

Entries are grouped into [sections].
IDs are assigned globally by order of appearance.

Entry forms inside a [section]:
  KEY = "string"      # Intern a string (quotes optional)
  KEY = @path         # Embed file bytes (path is relative to the .ini file)
  KEY = @@path        # Embed file bytes and append a null terminator. (path is relative to the .ini file)
```

---

# Overview

The `.ini` file describes **entries grouped by sections**.

Each entry produces a **stable numeric ID** and associated generated data in the header file.

IDs are assigned **globally in order of appearance** across all sections. :contentReference[oaicite:2]{index=2}

The generated header may include:

- numeric ID macros
- string constants
- embedded file byte arrays
- per-section enums
- per-section string arrays
- optional CRC checks

# Example

Example input file:

```
guard = DATA_H
output = data.h

[strings]
HELLO = "Hello world"
GOODBYE = "Goodbye"

[files]
ICON = @icon.png
SCRIPT = @@script.lua
```

Running:

```
bake data.ini
```

Generates a header file similar to:

```
#ifndef DATA_H
#define DATA_H

#define DATA_H_COUNT 3

#define STRINGS_HELLO 0
#define STRINGS_GOODBYE 1
#define FILES_ICON 2

...

#endif
```

Strings and files can then be accessed using the generated macros and arrays.

# Entry Types

Entries are declared inside sections.

```
KEY = VALUE
```

The interpretation of the value determines how the data is baked.

### String entries

```
KEY = "text"
```

Interns a string constant.

Quotes are optional.

Generated values include:

- string constant
- string length
- string hash

---

### File entries

```
KEY = @path
```

Embeds the contents of a file into the generated header as a byte array.

The path is **relative to the `.ini` file**. :contentReference[oaicite:3]{index=3}

Generated values include:

- file path
- file size
- byte array
- optional CRC

---

### Null-terminated file entries

```
KEY = @@path
```

Same as `@path`, but an extra `0` byte is appended to the data so it can be safely treated as a C string. :contentReference[oaicite:4]{index=4}

# INI File Options

These options appear at the **top level** of the `.ini` file (outside sections).

### guard

```
guard = DATA_H
```

Required.

Defines the include guard used in the generated header. :contentReference[oaicite:5]{index=5}

---

### output

```
output = generated.h
```

Optional output file name.

If not provided, the output file defaults to the `.ini` filename with `.h`. :contentReference[oaicite:6]{index=6}

---

### no_enum

```
no_enum = 0 | 1
```

If enabled, the generator **does not emit per-section enums**. :contentReference[oaicite:7]{index=7}

---

### no_strings

```
no_strings = 0 | 1
```

If enabled, the generator **does not emit string arrays for each section**. :contentReference[oaicite:8]{index=8}

---

### no_crc

```
no_crc = 0 | 1
```

If enabled, CRC32 hashes for embedded file contents are not generated. :contentReference[oaicite:9]{index=9}

---

### bytes_per_line

```
bytes_per_line = N
```

Controls how many bytes are written per line in generated byte arrays.

Default: `8`  
Maximum: `64`. :contentReference[oaicite:10]{index=10}

---

### comment

```
comment = "text"
```

Optional comment inserted at the top of the generated header.

Supports escape sequences such as:

```
\n
\t
``` 
:contentReference[oaicite:11]{index=11}

# Generated Data

For each entry the generator produces a set of macros and optionally arrays.

Typical output may include:

```
SECTION_KEY
SECTION_KEY_STR
SECTION_KEY_LEN
SECTION_KEY_HASH
SECTION_KEY_BYTES
SECTION_KEY_FILE_SIZE
SECTION_KEY_CRC
```

Files embedded with `@` or `@@` generate byte arrays containing the file contents.

---

# Typical Workflow

1. Create a `.ini` description file listing strings and files.
2. Run `bake` to generate a C header.
3. Include the generated header in your project.

```
bake assets.ini
```

4. Use the generated IDs and constants directly in code.

This allows assets, scripts, or static data to be embedded directly into the binary at compile time.
